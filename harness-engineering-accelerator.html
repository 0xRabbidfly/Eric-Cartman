03 — The Accelerator Package

Four Deployable Layers

Each layer can be delivered as a discrete engagement phase or bundled as a platform. The full stack converts isolated copilot use into a compounding delivery system.

1
Context Architecture & Platform Foundation
The harness infrastructure — tiered context, memory, guardrails

Tiered Context System

Layer 1: always-on architecture. Layer 2: per-task module docs. Layer 3: per-file history. Budget tracked per layer — alerts at 40% window.

Persistent Memory Files

decisions.md, failure-catalog.md, session-state.md. Agent reads at session start, writes at end. Append-only audit trail.

Guardrail Hierarchy

Hard limits (cost ceiling, protected files). Safety nets (test gates, lint). Golden paths (templates, handoffs). Audit log on every action.

Agent Specialization

Separate agents per domain: code-gen, testing, review, deployment. Scoped tool access. Structured handoffs. No universal tooling.

2
Workflow Integration & SDLC Harness Templates
Pre-built harnesses per service topology; feedforward + feedback across the lifecycle

Topology Templates

~80% of enterprise workloads fit ~5 topologies (CRUD API, event processor, data pipeline, UI service, ML inference). Pre-bundle harness per topology.

Feedforward Guides

Architecture principles, coding conventions, how-to skills, reference docs — formatted for LLM consumption. Inferential + computational.

Feedback Sensors

Pre-commit: linters, type-checkers, fast tests. CI pipeline: mutation testing, architectural drift detection. Runtime: SLO monitors, log anomaly agents.

Shift-Left Quality

Security, compliance, and accessibility checks embedded at the agent's generation step — not at the end of a release cycle. "Positive prompt injection" via linter messages.

3
Adoption Infrastructure & Internal Platform
The "Glass + Dojo" model — frictionless onboarding and a skills marketplace

Internal AI Platform

SSO-authenticated, auto-configured on install. 30+ enterprise tools light up immediately (Jira, Confluence, Salesforce, etc.). No terminal setup. If the user has to debug, you've already lost.

Skills Marketplace

Anyone packages a workflow and shares it org-wide. One team's breakthrough becomes every team's superpower. Ramp: 350+ skills in months.

L0→L3 Proficiency Ladder

Map where people are. Build tools to lift L0→L1 (low bar). Raise expectations as tools mature. Embed AI proficiency into hiring and performance language.

Hub + Spokes Org Model

Central team owns: platform, connectors, LLMs, data, knowledge, training. Functional teams build on top. Central roadmap driven by spokes. Avoids both centralization failure and redundant re-learning.

4
Governance, Measurement & Trust Layer
Making AI-native delivery safe enough for enterprise risk tolerance

AI-Specific KPIs

Supplement DORA metrics: agent task completion rate, harness sensor fire rate, context budget utilization, memory compounding rate, skills adoption velocity.

Agent Permissions Model

Risk-tiered autonomy: read-only at tier 1, write-with-review at tier 2, autonomous-with-audit at tier 3. Identity and accountability for every agent action.

Governance as Code

Policy and compliance controls embedded in harness sensors — not in separate review cycles. "Governance by design" per Forrester November 2025 guidance.

Evals Framework

Validate every skill and context file on real tasks. Unvalidated context becomes noise. Benchmarked, human-authored skills beat self-generated ones. Evals are table stakes for credibility.

06 — The Compounding Moat

Why the Harness Compounds and the Model Doesn't

🧠

Trajectories as the Dataset

Every agent execution through a well-instrumented harness generates structured, loggable, gradeable data. These trajectories are the fine-tuning input that makes your agents better over time — not someone else's.

📚

Failure Catalog Compounds

Each production failure gets indexed with its resolution pattern. Six months of accumulated failure catalog cannot be replicated by a competitor switching to your model or framework.

🔧

Skills Marketplace Flywheels

Each packaged skill raises the floor for every subsequent user. One team's breakthrough → 350+ shared skills. Knowledge compounds horizontally, not just vertically.

🌐

Domain Context Accumulates

Harness templates encode decisions, constraints, and architectural knowledge specific to your domain. A generic model cannot substitute for six months of enterprise-specific context engineering.

🏗️

Harnessability as a Design Choice

Architecture and language choices determine how governable a codebase will be. Teams that design for harnessability from day one will be able to deploy harness templates competitors with legacy systems cannot.

♻️

Self-Healing Loop

The harness improves every time it breaks: friction → workaround → skill reflection → fix applied → memory saved → harness stronger. The improvement loop is built into the delivery process.

07 — Delivery Playbook

Three Horizons for the Accelerator Engagement

HORIZON 1 · WEEKS 1–6
Foundation
  • Audit current tool deployment and stack health
  • Deploy internal platform (Glass model) with SSO + pre-wired connectors
  • Define 2–3 service topologies covering 80% of workloads
  • Build harness template for highest-value topology
  • Install basic feedback sensors (linters, type-checkers, fast tests)
  • Define L0→L3 proficiency map and baseline the team
10–15% Adoption floor. Teams producing. Baseline established.
HORIZON 2 · WEEKS 6–16
Workflow Redesign
  • Redesign 3–5 end-to-end SDLC workflows around agent loops
  • Deploy feedforward guides (architecture.md, how-to skills, domain docs)
  • Add CI pipeline sensors (mutation testing, architectural drift)
  • Launch skills marketplace; seed with 20+ validated skills
  • Define AI-specific KPI scorecard alongside DORA metrics
  • Establish agent permissions model by risk tier
25–30% Measured gains. Workflows compounding. Non-engineers building.
HORIZON 3 · WEEKS 16+
Autonomous Delivery
  • Expand harness templates to all major topologies
  • Deploy continuous runtime sensors (SLO monitors, log anomaly agents)
  • Implement eval framework validating all skills in production
  • Build failure catalog + decision log as institutional memory
  • Shift human review from code to intent, constraints, and escalation
  • Publish trajectories → fine-tuning feedback loop
40%+ Compounding moat. Self-improving harness. Governance as code.

09 — Design Questions

What Still Needs to Be Resolved Per Client

1

Where does human review move?

From PR-level code review → spec and intent review, acceptance criteria, risk-based escalation triggers. The transition speed depends on risk tier and team L3 density.

2

Unit of transformation: use case, workflow, or shared platform?

Isolated use cases compound slowly. Workflow redesign compounds faster. A shared enterprise platform compounds fastest — but requires Horizon 2–3 investment and organizational will.

3

What reusable IP does the firm build and own?

Options: prompts (lowest moat), skills/recipes, harness templates per topology, eval frameworks, domain-tuned models, or governed workflow orchestration. Each has different build-vs-buy curves.

4

How much autonomy is acceptable per workflow and risk tier?

A refactoring agent touching internal tooling ≠ a deployment agent touching production. Define the autonomy ladder before deploying agents, not after the first incident.

5

What is the apprenticeship path when agents do most implementation?

If junior developers spend less time writing code, what builds their judgment, pattern recognition, and accountability? The org model needs a new answer — harness template authorship is one candidate.

6

How do you keep the harness coherent as it grows?

Guides and sensors will drift, contradict each other, and bloat. Periodic "spa day" consolidation (prune stale rules, deduplicate sensors, re-eval skills) needs to be a scheduled ritual, not a crisis response.
